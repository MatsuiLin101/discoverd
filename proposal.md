# 旅遊業者網站開發提案

**提案日期：** 2026-05-27
**提案對象：** 找到了旅行社（小組）
**提案人：** MatsuiLin101

---

## 一、專案概述

本提案為貴小組規劃一套獨立的旅遊業者網站，功能參考同公司現有網站，採用現代化技術架構開發，版面設計與現有網站明顯區隔。

網站分為**前台**（供訪客瀏覽行程）與**後台**（供旅行社人員管理內容）兩大部分。

---

## 二、功能規格

### 導覽列
1. 目的地分類選單
2. 主題分類（暑期、連假等）
3. 行程關鍵字搜尋
4. 服務據點切換

### 首頁區塊
5. Hero Banner 輪播
6. 精選 / 熱門行程展示
7. 目的地分類格狀瀏覽
8. 每月限定優惠
9. 早鳥預售專區

### 行程相關
10. 行程列表頁
11. 行程詳情頁
12. 價格顯示（含航空公司資訊）
13. 地區 / 主題分類篩選

### 聯絡 / 互動
14. 浮動快捷按鈕（電話、Line、FB）
15. Line / Facebook / Instagram 連結

### 後台系統
16. 帳號登入（管理員 / 一般帳號兩種權限）
17. 同時間只允許一位使用者登入後台
18. 行程管理（新增 / 編輯 / 刪除）
19. 圖片上傳

### 其他
20. 頁尾公司資訊（執照、統編）
21. Google Analytics 4
22. 技術 SEO（Meta tags、Sitemap、robots.txt、結構化資料）

> **備註：** 本報價範圍僅含以上列出之 22 項功能，額外需求另行報價。

## 三、頁面架構

### 全站共用元素
- **導覽列**：Logo、目的地分類選單、主題分類選單、行程搜尋、服務據點
- **浮動快捷按鈕**：電話、Line、FB
- **頁尾**：公司資訊（執照、統編）、社群連結（Line / FB / Instagram）

### 頁面

```
首頁
├── Hero Banner 輪播

├── 目的地分類格狀瀏覽
├── 每月限定優惠

├── 價格資訊
```

## 四、設計風格方向

提供 **4 種風格方向**，各具備首頁與行程詳情頁的完整版面設計，供貴小組選擇。  
互動版預覽請開啟以下 HTML 檔案（以瀏覽器開啟即可）：

| 版本 | 風格名稱 | 首頁預覽 | 行程詳情預覽 |
|------|----------|----------|--------------|
| A | 編輯雜誌風 | `designs/Discovered_index_files/v2-a-magazine.html` | `designs/Discovered_tour_files/tour-a-magazine.html` |
| B | 側邊直書 | `designs/Discovered_index_files/v2-b-vertical.html` | `designs/Discovered_tour_files/tour-b-vertical.html` |
| C | 數字主張 | `designs/Discovered_index_files/v2-c-numbers.html` | `designs/Discovered_tour_files/tour-c-numbers.html` |
| D | Lookbook | `designs/Discovered_index_files/v2-d-lookbook.html` | `designs/Discovered_tour_files/tour-d-lookbook.html` |

### 風格說明

**A · 編輯雜誌風**
- 報紙分欄排版，文字與圖片並重
- 深酒紅配色，質感典雅
- 行程詳情呈現：詩意標題、多圖橫排、逐日行程列表、費用含不含清單

**B · 側邊直書**
- 全版大圖 Hero，側邊直書文字裝飾
- 日系侘寂美學，暗色系為主
- 行程詳情呈現：大圖開場、清晰的 N 日行程表、底部黑底報價欄

**C · 數字主張**
- 用數字說話（國家數、路線數、方案數）
- 深色系，專業信賴感強
- 行程詳情呈現：大數字統計欄、全寬 Hero、數據化行程表

**D · Lookbook**
- 全螢幕沉浸式，目的地以大圖依序滾動呈現
- 極簡奢華，圖片主導
- 行程詳情呈現：全版圖片疊字，沉浸式閱讀體驗

### 共同設計元素（四款通用）
- 品牌色：深玫瑰紅（`#D12351`）
- 字體：Noto Sans TC（繁體中文優化）
- 版面與現有 discovered.tw 明顯區隔

## 五、技術規格

| 項目 | 技術選型 |
|------|----------|
| 前台框架 | Next.js（React）— 支援 SSR，對 SEO 友好 |
| 後台 API | Next.js API Routes — 與前台共用一個專案 |
| 資料庫 | PostgreSQL（透過 Supabase 托管） |
| 圖片儲存 | Cloudinary |
| 後台單一登入 | JWT + Server-side Session，登入時記錄使用者，登出或心跳中斷時清除 |
| 帳號權限 | 管理員可管理帳號與行程；一般帳號僅可管理行程 |

### 使用量監控通知

各服務均設定使用量達 **80%** 時自動發送 Email 通知，讓維護人員提前評估是否需要升級：

| 服務 | 監控項目 |
|------|----------|
| Vercel | 流量、函式執行量 |
| Supabase | 資料庫容量、連線數、頻寬 |
| Cloudinary | 儲存空間、月流量 |
| Cloudflare | 流量異常、錯誤率 |

### 流量承載能力

- **前台靜態頁面**：透過 Cloudflare CDN 分發，可支撐數萬人同時瀏覽
- **動態搜尋 / 篩選**：可支撐數百至一千個同時請求
- **後台**：設計上限制單人使用，不受流量影響

### 未來擴充

若流量持續成長需升級，可評估遷移至單一雲端主機（如 AWS、GCP），遷移工作約需 1–3 天，不影響網站原始碼結構。
