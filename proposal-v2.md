**提案日期：** 2026-05-27
**提案對象：** 找到了旅行社（小組）
**提案人：** MatsuiLin101

## 一、專案概述

本提案為貴小組規劃一套獨立的旅遊業者網站，採用現代化技術架構開發，版面設計與現有網站明顯區隔。

**行程內容管理方式**，本提案提供三種方案供選擇（報價詳見第八節）：

| 方案 | 說明 | 開發時程 | 報價 |
|------|------|----------|------|
| A · 行程圖片上傳 | 將行程 PDF 轉為圖片後上傳，網頁直接顯示圖片 | 2 週 | NT$45,000 |
| B · Rich Text 編輯器 | 行程內容在後台 CMS 直接編輯（逐日行程、費用含不含），完整自訂樣式 | 2.5 週 | NT$45,000 |
| C · 兩者皆有 | 同時支援圖片上傳與 Rich Text 編輯，可混用或逐步遷移 | 3 週 | NT$50,000 |

## 二、功能規格

### 導覽列
1. 目的地分類選單（兩層：大區 → 小區）
2. 行程關鍵字搜尋（全站）
3. 服務據點切換（Modal，含地圖導航連結）
4. 動態麵包屑導覽

### 首頁
5. 目的地大分類格狀瀏覽（13 個目的地，如日本、韓國、歐洲…）

### 行程相關
6. 目的地小分類格狀瀏覽（如日本 → 東京、關西、北海道…）
7. 行程列表頁（含 tag 標籤、門市價格、航空公司）
8. 行程詳情（價格 + 聯絡資訊；行程內容依選擇方案呈現，見下方說明）
9. 行程分享連結（複製當前行程 URL）

### 後台系統
10. 帳號登入（管理員 / 一般帳號兩種權限）
11. 同時間只允許一位使用者登入後台
12. 行程管理（新增 / 編輯 / 刪除）
    - 基本欄位（三方案共用）：行程名稱、目的地分類、標籤（熱門 / 超值…）、門市價格、縮圖
    - 方案 A 額外欄位：行程圖片上傳（可多張）
    - 方案 B 額外欄位：Rich Text 編輯器（逐日行程、費用含不含）
    - 方案 C 額外欄位：行程圖片上傳 + Rich Text 編輯器（兩者並存）

### 其他
13. 頁尾公司資訊（執照、統編）
14. Google Analytics 4
15. 技術 SEO（Meta tags、Sitemap、robots.txt、結構化資料）
16. SSL 憑證（Cloudflare 配發，自動免費）

> **備註：** 本報價範圍僅含以上列出之 16 項功能，額外需求另行報價。

## 三、頁面架構

### 全站共用元素
- **導覽列**：Logo、目的地分類選單（兩層）、行程搜尋、服務據點、麵包屑
- **頁尾**：公司資訊（執照、統編）

### 頁面（SPA 架構，無整頁重新載入）

```
首頁
└── 目的地大分類格狀瀏覽

目的地小分類頁（如點選「日本」後顯示）
└── 小分類格狀瀏覽（東京、關西、北海道…）

行程列表頁（點選小分類後顯示）
├── 行程卡片列表（含 tag、價格、航空公司）
└── 關鍵字搜尋結果也導向此頁

行程詳情（Modal，不離開當前頁面）
├── 行程內容（依方案：圖片 / Rich Text / 兩者）
├── 行程名稱、tag、價格
├── 行程分享連結（複製當前行程 URL）
└── 門市聯絡資訊

服務據點（Modal）
├── 各門市：名稱、電話、地址
├── 地圖導航（Google Maps 連結）
└── 切換門市按鈕
```

### 頁面數量總覽

| 類別 | 頁面 | 數量 |
|------|------|------|
| 前台 | 首頁、目的地小分類頁、行程列表頁 | 3 頁 |
| 前台 Modal | 行程詳情、服務據點 | 2 個 |
| 後台 | 登入頁、行程管理頁、帳號管理頁 | 3 頁 |
| **合計** | | **6 頁 + 2 Modal** |

> Modal 不跳轉頁面，在當前頁面覆蓋開啟。

## 四、設計風格方向

提供 **4 種風格方向**，各具備首頁與行程詳情頁的完整版面設計，供貴小組選擇。  
互動版預覽請開啟以下 HTML 檔案（以瀏覽器開啟即可）：

| 版本 | 風格名稱 | 首頁預覽 | 行程詳情預覽 |
|------|----------|----------|--------------|
| A | 編輯雜誌風 | `designs/Discovered_index_files/v2-a-magazine.html` | `designs/Discovered_tour_files/tour-a-magazine.html` |
| B | 側邊直書 | `designs/Discovered_index_files/v2-b-vertical.html` | `designs/Discovered_tour_files/tour-b-vertical.html` |
| C | 數字主張 | `designs/Discovered_index_files/v2-c-numbers.html` | `designs/Discovered_tour_files/tour-c-numbers.html` |
| D | Lookbook | `designs/Discovered_index_files/v2-d-lookbook.html` | `designs/Discovered_tour_files/tour-d-lookbook.html` |

### 風格說明

**A · 編輯雜誌風**
- 報紙分欄排版，文字與圖片並重
- 深酒紅配色，質感典雅
- 行程詳情呈現：詩意標題、多圖橫排、逐日行程列表、費用含不含清單

**B · 側邊直書**
- 全版大圖 Hero，側邊直書文字裝飾
- 日系侘寂美學，暗色系為主
- 行程詳情呈現：大圖開場、清晰的 N 日行程表、底部黑底報價欄

**C · 數字主張**
- 用數字說話（國家數、路線數、方案數）
- 深色系，專業信賴感強
- 行程詳情呈現：大數字統計欄、全寬 Hero、數據化行程表

**D · Lookbook**
- 全螢幕沉浸式，目的地以大圖依序滾動呈現
- 極簡奢華，圖片主導
- 行程詳情呈現：全版圖片疊字，沉浸式閱讀體驗

### 共同設計元素（四款通用）
- 品牌色：深玫瑰紅（`#D12351`）
- 字體：Noto Sans TC（繁體中文優化）
- 版面與現有 discovered.tw 明顯區隔

> **關於行程詳情頁的設計說明：** 設計稿中的行程詳情內容排版（逐日行程、費用含不含等）以方案 B / C 為基準。選擇方案 A 時，相同的頁面框架與風格不變，行程內容區域改以上傳的圖片呈現。

## 五、技術規格

| 項目 | 技術選型 |
|------|----------|
| 前台框架 | Next.js（React）— 支援 SSR，對 SEO 友好 |
| 後台 API | Next.js API Routes — 與前台共用一個專案 |
| 資料庫 | PostgreSQL（透過 Supabase 托管） |
| 行程內容（方案 A） | 圖片上傳（Cloudinary），將行程 PDF 轉圖後上傳，支援多張 |
| 行程內容（方案 B） | Tiptap Rich Text 編輯器，內容存於資料庫 |
| 行程內容（方案 C） | 同時支援圖片上傳與 Rich Text，後台可混用 |
| 縮圖儲存 | Cloudinary |
| 後台單一登入 | JWT + Server-side Session，登入時記錄使用者，登出或心跳中斷時清除 |
| 帳號權限 | 管理員可管理帳號與行程；一般帳號僅可管理行程 |

### 行程內容方案比較

| | 方案 A · 圖片上傳 | 方案 B · Rich Text | 方案 C · 兩者皆有 |
|---|---|---|---|
| 行程維護方式 | 將 PDF 轉圖後上傳 | 後台 CMS 編輯 | 兩者均可 |
| 手機體驗 | 好 | 好 | 好 |
| SEO | 無（圖片內容不被收錄） | 完整 | 完整 |
| 學習成本 | 低 | 中（需學後台編輯器） | 中 |
| 彈性 | 中（版面固定於圖片設計） | 高 | 最高 |
