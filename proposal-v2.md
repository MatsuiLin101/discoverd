# 旅遊業者網站開發提案（精簡版）

**提案日期：** 2026-05-27
**提案對象：** 找到了旅行社（小組）
**提案人：** MatsuiLin101
**版本：** V2 精簡版（對應 discovered.tw 現有功能）

---

## 一、專案概述

本提案為貴小組規劃一套獨立的旅遊業者網站，功能規格對應同公司現有 discovered.tw 的實際功能，採用現代化技術架構開發，版面設計與現有網站明顯區隔。

與 V1 完整版的主要差異：
- **行程內容改用 Google Drive 嵌入**，不建自訂 Rich Text 編輯器
- 首頁簡化為目的地分類格狀瀏覽，移除 Hero Banner、每月優惠、早鳥預售等區塊
- 移除主題分類、浮動社群按鈕
- 後台職責縮減：僅管理行程清單（名稱、分類、價格、縮圖、連結）

---

## 二、功能規格

### 導覽列
1. 目的地分類選單（兩層：大區 → 小區）
2. 行程關鍵字搜尋（全站）
3. 服務據點切換（Modal，含地圖導航連結）
4. 動態麵包屑導覽

### 首頁
5. 目的地大分類格狀瀏覽（13 個目的地，如日本、韓國、歐洲…）

### 行程相關
6. 目的地小分類格狀瀏覽（如日本 → 東京、關西、北海道…）
7. 行程列表頁（含 tag 標籤、門市價格、航空公司）
8. 行程詳情（嵌入 Google Drive 文件 + 價格 + QR Code + 聯絡資訊）
9. 行程分享連結（複製當前行程 URL）

### 後台系統
10. 帳號登入（管理員 / 一般帳號兩種權限）
11. 同時間只允許一位使用者登入後台
12. 行程管理（新增 / 編輯 / 刪除）
    - 管理欄位：行程名稱、目的地分類、標籤（熱門 / 超值…）、門市價格、縮圖、Google Drive 連結

### 其他
13. 頁尾公司資訊（執照、統編）
14. Google Analytics 4
15. 技術 SEO（Meta tags、Sitemap、robots.txt、結構化資料）
16. SSL 憑證（Cloudflare 配發，自動免費）

> **備註：** 本報價範圍僅含以上列出之 16 項功能，額外需求另行報價。

## 三、頁面架構

### 全站共用元素
- **導覽列**：Logo、目的地分類選單（兩層）、行程搜尋、服務據點、麵包屑
- **頁尾**：公司資訊（執照、統編）

### 頁面（SPA 架構，無整頁重新載入）

```
首頁
└── 目的地大分類格狀瀏覽

目的地小分類頁（如點選「日本」後顯示）
└── 小分類格狀瀏覽（東京、關西、北海道…）

行程列表頁（點選小分類後顯示）
├── 行程卡片列表（含 tag、價格、航空公司）
└── 關鍵字搜尋結果也導向此頁

行程詳情（Modal，不離開當前頁面）
├── Google Drive 文件嵌入（主體）
├── 行程名稱、tag、價格
├── QR Code（連到 Google Drive）
└── 門市聯絡資訊

服務據點（Modal）
├── 各門市：名稱、電話、地址
├── 地圖導航（Google Maps 連結）
└── 切換門市按鈕
```

共 **2 個實體頁面**（首頁 + 小分類/列表頁）+ 2 個 Modal + 全站共用元素。

## 五、技術規格

| 項目 | 技術選型 |
|------|----------|
| 前台框架 | Next.js（React）— 支援 SSR，對 SEO 友好 |
| 後台 API | Next.js API Routes — 與前台共用一個專案 |
| 資料庫 | PostgreSQL（透過 Supabase 托管） |
| 行程內容 | Google Drive 嵌入（iframe），行程 PDF / Slides 由客戶自行維護 |
| 縮圖儲存 | Cloudinary |
| 後台單一登入 | JWT + Server-side Session，登入時記錄使用者，登出或心跳中斷時清除 |
| 帳號權限 | 管理員可管理帳號與行程；一般帳號僅可管理行程 |

### 行程內容維護說明

本方案的行程詳情使用 **Google Drive 嵌入**，意即：
- 行程 PDF 或投影片由旅行社人員在 Google Drive 自行更新
- 後台只需填入 Google Drive 分享連結，網站即自動同步最新內容
- 不需額外登入後台，客戶熟悉的工具零學習成本
- 缺點：行程詳情的版面由 Google Drive 呈現，無法自訂樣式
