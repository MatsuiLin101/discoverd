# 旅遊業者網站開發提案

**提案日期：** 2026-06-01
**提案對象：** 找到了旅行社（小組）
**提案人：** MatsuiLin101

## 一、專案概述

本提案為貴小組規劃一套獨立的旅遊業者網站，採用現代化技術架構開發，版面設計與現有網站明顯區隔。

**行程內容管理方式**，本提案提供三種方案供選擇（報價詳見第八節）：

| 方案 | 說明 | 開發時程 | 報價 |
|------|------|----------|------|
| A · 多檔上傳 | 同時上傳多個 PDF 或圖片檔，網頁以連續圖片方式顯示 | 2 週 | NT$45,000 |
| B · Rich Text 編輯器 | 行程內容在後台 CMS 直接編輯（逐日行程、費用含不含），完整自訂樣式 | 2.5 週 | NT$45,000 |
| C · 兩者皆有 | 同時支援多檔上傳與 Rich Text 編輯，可混用或逐步遷移 | 3 週 | NT$50,000 |

## 二、功能規格

### 導覽列
1. 目的地分類選單（兩層：大區 → 小區）
2. 行程關鍵字搜尋（含目的地分類篩選、即時搜尋建議）
3. 動態麵包屑導覽
4. 社群媒體連結（Facebook、Instagram、LINE）

### 首頁
5. 首頁輪播大圖（每隔數秒自動切換目的地圖片，含左右切換箭頭）
6. 目的地大分類格狀瀏覽（含縮圖、路線數量）

### 行程相關
7. 目的地小分類格狀瀏覽（含縮圖、路線數量）
8. 行程列表頁（含小分類頁籤切換、tag 標籤、行程價格）
9. 行程詳情（Modal）：行程圖片 / 內容、名稱、tag、價格
10. LINE 諮詢按鈕（導向品牌 LINE 帳號）
11. 線上諮詢表單（Modal：姓名、行動電話、Email、LINE ID、諮詢內容）
12. 行程分享連結（複製當前行程 URL）

### 後台系統
13. 帳號登入（管理員 / 一般帳號兩種權限）
14. 同時間只允許一位使用者登入後台
15. 行程管理（新增 / 編輯 / 刪除）
    - 基本欄位（三方案共用）：行程名稱、目的地分類、標籤（熱門 / 超值…）、行程價格、縮圖
    - 方案 A 額外欄位：行程檔案上傳（可同時上傳多個 PDF 或圖片檔）
    - 方案 B 額外欄位：Rich Text 編輯器（逐日行程、費用含不含）
    - 方案 C 額外欄位：行程檔案上傳 + Rich Text 編輯器（兩者並存）

### 其他
16. 頁尾公司資訊（執照、統編）
17. Google Analytics 4
18. 技術 SEO（Meta tags、Sitemap、robots.txt、結構化資料）
19. SSL 憑證（Cloudflare 配發，自動免費）

> **備註：** 本報價範圍僅含以上列出之 19 項功能，額外需求另行報價。

## 三、頁面架構

### 全站共用元素
- **導覽列**：Logo、目的地分類選單（兩層）、行程搜尋（含分類篩選）、社群媒體連結、麵包屑
- **頁尾**：公司資訊（執照、統編）

### 頁面

```
首頁
├── 輪播大圖（每隔數秒自動切換）
└── 目的地大分類格狀瀏覽

目的地小分類頁（點選大分類後顯示）
└── 小分類格狀瀏覽（如日本 → 北海道、東京、關西…）

行程列表頁（點選小分類後顯示）
├── 小分類頁籤（同頁切換，不重新載入）
└── 行程卡片列表（含 tag、行程價格）

行程詳情（Modal，不離開當前頁面）
├── 行程內容（依方案：圖片 / Rich Text / 兩者）
├── 行程名稱、tag、價格
├── LINE 諮詢按鈕
├── 線上諮詢表單（按鈕開啟子 Modal）
└── 行程分享連結（複製當前行程 URL）

線上諮詢表單（子 Modal，疊於行程詳情之上）
├── 聯絡人、行動電話（必填）
├── Email、LINE ID（選填）
├── 諮詢內容（必填）
└── 送出 → 顯示確認訊息
```

### 頁面數量總覽

| 類別 | 頁面 | 數量 |
|------|------|------|
| 前台 | 首頁、目的地小分類頁、行程列表頁 | 3 頁 |
| 前台 Modal | 行程詳情、線上諮詢表單 | 2 個 |
| 後台 | 登入頁、行程管理頁、帳號管理頁 | 3 頁 |
| **合計** | | **6 頁 + 2 Modal** |

> Modal 不跳轉頁面，在當前頁面覆蓋開啟。

## 四、設計定稿

設計風格已確認，採用首頁輪播大圖搭配毛玻璃效果導覽列的現代簡約風格，配色以珊瑚玫瑰紅（`#E0506B`）與薰衣草紫（`#7166D6`）為主，輕盈漸層背景貫穿全站。

## 五、技術規格

| 項目 | 技術選型 |
|------|----------|
| 前台框架 | Next.js（React）— 支援 SSR，對 SEO 友好 |
| 後台 API | Next.js API Routes — 與前台共用一個專案 |
| 資料庫 | PostgreSQL（透過 Supabase 托管） |
| 行程內容（方案 A） | 多檔上傳（PDF 或圖片），可同時選取多個檔案，統一儲存至 Cloudinary |
| 行程內容（方案 B） | Tiptap Rich Text 編輯器，內容存於資料庫 |
| 行程內容（方案 C） | 同時支援多檔上傳與 Rich Text，後台可混用 |
| 縮圖儲存 | Cloudinary |
| 後台單一登入 | JWT + Server-side Session，登入時記錄使用者，登出或心跳中斷時清除 |
| 帳號權限 | 管理員可管理帳號與行程；一般帳號僅可管理行程 |
| 線上諮詢表單 | 送出後以 Email 通知管理員（Resend，免費方案） |

### 行程內容方案比較

| | 方案 A · 多檔上傳 | 方案 B · Rich Text | 方案 C · 兩者皆有 |
|---|---|---|---|
| 行程維護方式 | 直接上傳 PDF 或圖片（可多檔同時上傳） | 後台 CMS 編輯 | 兩者均可 |
| 手機體驗 | 好 | 好 | 好 |
| SEO | 無（圖片內容不被收錄） | 完整 | 完整 |
| 學習成本 | 低 | 中（需學後台編輯器） | 中 |
| 彈性 | 中（版面固定於上傳檔案） | 高 | 最高 |

| 第 3 週（前半） | 前台頁面開發、GA4 整合、整合測試、部署上線 |
